# obsidian-task-center

用于在本地 Obsidian vault 中建立并维护可复用的任务管理与知识沉淀结构。

## 1) 适用场景

当用户提到以下需求时，优先使用此技能：

- 优化 Obsidian 里的任务中心
- 给任务系统加模板 / 详情页 / 总览页
- 新增“经验”“技能”目录并接入任务流
- 让任务完成后自动沉淀经验 / 技能
- 调整总览页开头的概览样式（适配 Obsidian 悬停预览）
- 清理 Obsidian vault 中无关文件、空目录（需保守处理）

## 2) 核心目标

把 Obsidian 里的工作流整理成三段式闭环：

1. **任务中心**：记录任务状态、详情、归档
2. **经验**：记录做完后学到了什么、踩了哪些坑
3. **技能**：记录以后怎么稳定复用、标准做法是什么

目标不是只做目录整理，而是让结构能长期维护。

## 3) 标准目录结构

推荐至少包含：

```text
任务中心/
  00-任务总览.md
  10-进行中任务.md
  20-长期任务.md
  30-已完成任务.md
  90-任务模板.md
  99-使用说明.md
  详情-*.md
经验/
  00-经验总览.md
  xxx经验.md
技能/
  00-技能总览.md
  xxx技能.md
```

## 4) 总览页设计原则

### 4.1 悬停预览优先

Obsidian 中把鼠标悬停在链接上时，会优先显示文档前几行。

因此：
- 文档开头优先放 **紧凑表格摘要**
- 不要先放大段说明
- 让用户不点进去也能知道：
  - 有哪些任务 / 条目
  - 当前状态
  - 最近更新时间

### 4.2 推荐表格字段

#### 任务总览

| 分类 | 入口 | 概览 | 最近更新时间 |
|---|---|---|---|

#### 任务列表页

| 任务 | 状态 | 概览 | 最近更新时间 |
|---|---|---|---|

#### 经验 / 技能总览

| 名称 | 概览 | 最近更新时间 |
|---|---|---|

### 4.3 概览字段要求

- 尽量短
- 总览页里只写状态级概览即可
- 例如：
  - `1 项进行中`
  - `1 项长期进行中`
  - `2 项经验`
  - `2 项技能`
- 任务页里再写更具体但仍简洁的概览

## 5) 任务设计原则

### 5.1 三层结构

推荐把任务拆成三层：

1. **总览页**：整体导航
2. **任务列表页**：进行中 / 长期 / 已完成
3. **详情页**：复杂任务的背景、步骤、风险、验收、复盘

### 5.2 复杂任务要补详情页

适用于：
- 多步骤任务
- 长期任务
- 需要经验 / 技能沉淀的任务
- 需要记录风险、验收标准的任务

### 5.3 任务条目推荐字段

| 字段 | 说明 |
|---|---|
| 状态 | 当前状态 |
| 优先级 | 高 / 中 / 低 |
| 创建时间 | 任务创建时间 |
| 最近更新时间 | 最近一次更新 |
| 当前进展 | 任务目前到哪一步 |
| 详情文档 | 对应详情页 |
| 相关经验 | 对应经验文档 |
| 相关技能 | 对应技能文档 |
| 验收标准 | 完成判断 |
| 备注 | 其他补充 |

## 6) 经验 / 技能接入规则

### 6.1 经验文档

记录：
- 本次踩坑
- 解决方法
- 后续标准做法
- 可关联任务

### 6.2 技能文档

记录：
- 适用场景
- 前提条件
- 标准步骤
- 常见故障
- 相关任务

### 6.3 闭环要求

后续所有任务，尽量挂上：
- 详情文档
- 相关经验
- 相关技能

任务完成后，必须检查是否需要：
- 新建经验文档
- 优化已有经验文档
- 新建技能文档
- 优化已有技能文档

## 7) 模板策略

不建议把一次性任务、长期任务、详情页模板拆成多个零散文件。

推荐统一放在：

```text
任务中心/90-任务模板.md
```

建议至少包含：
- 模板 A：一次性 / 阶段性任务条目模板
- 模板 B：长期 / 重复任务模板
- 模板 C：任务详情页模板

## 8) 清理策略

清理 vault 时要保守：

### 可以直接清理
- 已确认的空目录
- 已废弃且已被合并替代的模板文件

### 不要直接清理
- 用途不明的笔记
- 根目录下看不懂的文件
- 可能仍在被链接引用的页面

推荐流程：
1. 先列清单
2. 分成“保留 / 建议归档 / 疑似可删”
3. 经用户确认后再删不确定内容

## 9) 输出风格

- 先给结果，不要先讲一堆原则
- 对 Obsidian 用户优先优化“可悬停阅读性”
- 说明要短，结构要稳，链接要够用

## 10) 交付建议（便于分享给别的 OpenClaw）

若要把本技能直接分享给别的 OpenClaw，建议至少一起提供：

- `SKILL.md`
- `references/example-templates.md`
- `references/sample-vault-structure.md`
- `references/sample-task-pack.md`
- `references/cleanup-guidelines.md`

这样对方不仅知道原则，还能直接照着最小结构和样例任务落地。

## 11) 与其他技能的关系

- 若用户要打开 vault / 打开笔记 / 生成 URI：优先配合 `obsidian-cli`
- 若用户要整理任务系统 / 模板 / 经验技能结构：优先使用本技能
